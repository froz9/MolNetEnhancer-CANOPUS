# MolNetEnhancer-CANOPUS
An interactive R Shiny application designed to advance the MolNetEnhancer workflow for GNPS2 molecular networking data.

This tool bypasses traditional annotation bottlenecks in untargeted metabolomics by propagating deep-learning-based CANOPUS (SIRIUS) chemical class predictions across molecular families. It features a novel consensus algorithm that establishes high-confidence structural "hotspots" via live NPClassifier API queries, making it highly effective for annotating "dark matter" in complex biological or marine science samples.

✨ Key Features
GNPS2 Native Compatibility: Automatically parses GNPS2 .graphml files and calculates missing topological parameters (like componentindex) on the fly.

SIRIUS / CANOPUS Integration: Merges and propagates in silico chemical classifications (Superclass, Class, Subclass) across entire molecular families.

Empirical Hotspot Weighting: Extracts library_SMILES from GNPS2 matches and queries the NPClassifier API in real-time. These empirical hits are weighted dynamically to anchor the molecular family's consensus.

Custom Tie-Breaker Logic: Resolves classification conflicts in small molecular families (e.g., 2-node networks) by allowing users to choose between a "Classic" structural default or an "Ambiguous/Concatenated" output.

Clean Exports: Generates a lightweight summary .csv and injects the new consensus classifications directly into a downloadable, publication-ready .graphml file.

🛠️ Prerequisites & Installation
This application requires R and the Shiny web framework.

To run the app locally, ensure you have the following R packages installed:

R
install.packages(c("shiny", "igraph", "dplyr", "httr", "jsonlite", "DT"))
Running the App
Clone this repository to your local machine.

Open app.R in RStudio.

Click Run App in the top right corner of the script editor, or execute:

R
shiny::runApp("path/to/MolNetConsensus")
🚀 Usage Guide
Upload GNPS2 Network: Provide the .graphml file downloaded directly from your GNPS2 task.

Upload CANOPUS Summary: Provide the standard .tsv summary file generated by SIRIUS/CANOPUS.

Select Tie-Breaker Method:

Classic (First Annotation): Resolves ties by selecting the first processed annotation (mirrors the original pyMolNetEnhancer logic).

Ambiguous (Concatenate Ties): Safely flags ties by concatenating conflicting classes (e.g., Lipid / Alkaloid), preventing the loss of biological nuance.

Run Analysis: The app will parse the network, ping the NPClassifier API for library hotspots, and propagate the consensus.

Download: Export the summarized feature .csv or the fully annotated .graphml network for visualization in Cytoscape.

🧠 The Consensus Algorithm
MolNetConsensus uses a weighted frequency algorithm to determine the dominant chemical class within a connected component (molecular family):

Hotspot Identification: Nodes with a valid GNPS2 library_SMILES are queried against the NPClassifier API.

Weighted Voting: * A classification derived from a Library Hotspot is assigned a Weight = 2.

A classification derived from a CANOPUS Prediction is assigned a Weight = 1.

Propagation: The algorithm calculates the highest accumulated weight per component. If a node lacks a direct annotation, it inherits the winning consensus class of its molecular family.

Singleton Protection: Unconnected nodes (singletons, componentindex = -1) do not participate in consensus polling and retain their original CANOPUS classification with a defaulted 100% confidence score.
